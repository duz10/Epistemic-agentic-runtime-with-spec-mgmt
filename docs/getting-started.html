Agentic IDE

Cursor, Windsurf, VS Code + Copilot

Best for: visual learners, teams already in an IDE, designers who code.

  • ELCS loads automatically via .cursorrules or .windsurfrules
  • Inline AI chat reads ELCS state
  • No terminal required for basics

Bootstrap file: .cursorrules / .windsurfrules / AGENTS.md

‚å®Ô∏è

CLI / Terminal

Claude Code CLI, Codex, any terminal AI

Best for: developers comfortable in the terminal, automation-focused workflows, CI/CD integration.

  • Agent reads CLAUDE.md or AGENTS.md on start
  • Full control over agent context
  • Great for scripted and headless workflows

Bootstrap file: CLAUDE.md / AGENTS.md

üê∂

Code Puppy (Advanced)

Open-source agentic framework with ELCS-aware plugins

Best for: power users, teams wanting full observability, multi-agent coordination.

  • ELCS-aware out of the box via AGENTS.md
  • Optional telemetry plugin for passive observability
  • Multi-agent delegation with sub-agents
  • Custom plugin system for extending workflows

Bootstrap file: AGENTS.md + telemetry plugin

The key insight: ELCS uses simple markdown and JSON files. Any AI that can read files can follow the protocol. The dev mode you choose affects your experience ‚Äî not the framework's capabilities.

üöÄ Installation

Four ways to get started ‚Äî from copy-paste simple to CLI-powered.

üå± Beginner-friendly path

Step 1: Get the ELCS template

Open a terminal (on Windows: search for "PowerShell"; on Mac: search for "Terminal") and run:

git clone https://github.com/duz10/Epistemic-agentic-runtime-with-spec-mgmt.git

This downloads the ELCS framework to your computer. You only need to do this once.

Step 2: Create your project

Copy the template folder and rename it to your project name:

# Mac/Linux:
cp -r Epistemic-agentic-runtime-with-spec-mgmt/template my-project

# Windows PowerShell:
Copy-Item -Recurse Epistemic-agentic-runtime-with-spec-mgmt\template my-project

Now you have a my-project folder with all the ELCS files ready to go.

Step 3: Open in your AI tool

Open the folder in whichever tool you chose in "Choose Your Dev Mode":

# Cursor:
cd my-project && cursor .

# VS Code:
cd my-project && code .

# Claude Code CLI:
cd my-project && claude

# Code Puppy:
cd my-project && uvx code-puppy

The AI will automatically detect ELCS and start following the protocol. You're ready to build!

‚ö° Your First Session

Once you've got the template, here's what your first session looks like.

Step 1 ‚Äî Open & Orient

Open the project folder in your AI tool (Cursor, Claude, Windsurf, etc.). The agent reads the bootstrap file (CLAUDE.md, .cursorrules, etc.) and understands ELCS automatically.

"Please help me initialize this ELCS project.
Ask me about my goals and create
the initial state."

Step 2 ‚Äî Capture Beliefs

The agent helps you articulate what you believe, assume, and know. These become your Epistemic State ‚Äî stored in files, not chat memory.

# elcs/state/current.json
{
  "assumptions": [...],
  "hypotheses": [...],
  "constraints": [...]
}

Step 3 ‚Äî Apply Lenses

The agent evaluates your project from 7 perspectives, identifying risks, gaps, and open questions. Each lens produces structured output with actionable items.

"Apply the 7 lenses to our current
state and identify gaps."

Step 4 ‚Äî Earn Goals & Build

Goals pass through the 6 gates. Approved goals become WorkTokens. The agent executes Ralph loops, checkpoints regularly, and you review at natural breakpoints.

"Check elcs/tokens/open/ and
recommend the highest priority task."

The one rule: Write to files, not just chat. If your session ends, ELCS artifacts survive. That's the entire philosophy in one sentence.
