üöÄ Start a New Project

Three steps. Works for beginners and pros alike.

Step 1: Download ELCS

Open a terminal (Windows: search "PowerShell" ¬∑ Mac: search "Terminal") and run:

git clone https://github.com/duz10/Epistemic-agentic-runtime-with-spec-mgmt.git

This downloads the ELCS framework to your computer. You only do this once.

Step 2: Create your project from the template

# Mac / Linux:
cp -r Epistemic-agentic-runtime-with-spec-mgmt/template my-project

# Windows PowerShell:
Copy-Item -Recurse Epistemic-agentic-runtime-with-spec-mgmt\template my-project

Replace my-project with whatever you want to call it. You now have a folder with all the ELCS files ready to go.

Step 3: Open it in your AI tool and start talking

Navigate into your project and open it with your chosen dev mode:

# Cursor:
cd my-project && cursor .

# VS Code:
cd my-project && code .

# Claude Code CLI:
cd my-project && claude

# Code Puppy:
cd my-project && uvx code-puppy

Now just describe your project naturally. Tell the agent:

  • What you want to build
  • What problem you're solving
  • Any technical preferences or constraints
  • Or just think out loud ‚Äî ELCS captures it all

The agent automatically detects ELCS, reads the protocol, and walks you through the process ‚Äî capturing your goals, assumptions, and evidence as files along the way.

That's it. You don't need to understand every ELCS concept to start. Just describe what you want, and the framework guides both you and the agent through structured, persistent collaboration.

üîÑ Add ELCS to an Existing Project

Already have a codebase? You can bring ELCS into it without restructuring anything.

Pre-requisites

  • You've already cloned the ELCS framework repo (see Step 1 above)
  • You have an existing project folder you want to make ELCS-compliant

Option A: Use the install script (automated)

The install script copies bootstrap files into your project without touching your existing code:

# Mac / Linux:
./Epistemic-agentic-runtime-with-spec-mgmt/install.sh ~/path/to/my-project

# Windows PowerShell:
.\Epistemic-agentic-runtime-with-spec-mgmt\install.ps1 -TargetDir C:\path\to\my-project

This adds CLAUDE.md, AGENTS.md, .cursorrules, and reference files. It does not create the elcs/ folder yet ‚Äî your agent does that next.

Option B: Let your agent do it (conversational)

Open your existing project in your AI tool and simply tell it:

"I want to adopt the ELCS framework for this project.
Reference the ELCS protocol in
[path to your cloned ELCS repo]
and make this project fully compliant."

The agent will:

  • Read your existing codebase
  • Create the elcs/ folder with inferred state
  • Populate assumptions, constraints, and evidence from what already exists
  • Set up bootstrap files for your dev mode

You review and confirm, then continue working with full ELCS support going forward.

No restructuring required. ELCS lives in its own elcs/ folder and doesn't touch your existing code, tests, or docs. It's additive ‚Äî not invasive.

‚ö° Your First Session

Once you've got the template, here's what your first session looks like.

Step 1 ‚Äî Open & Orient

Open your project folder in your AI tool. The agent reads the bootstrap file (CLAUDE.md, .cursorrules, etc.) and understands ELCS automatically.

Already have a plan? Point the agent to it:

"Look at my detailed plan in
docs/project-plan.md and help
me get started with ELCS."

Don't have a plan yet? Just start talking about your idea ‚Äî ELCS builds the plan as you go:

"I want to build an app that
tracks my groceries from receipt
photos. I'm not sure where to
start ‚Äî help me think through it."

Step 2 ‚Äî Capture Beliefs

As you talk, the agent captures what you believe, assume, and know into your Epistemic State ‚Äî stored in files, not chat memory.

The agent tracks everything dynamically as you work ‚Äî assumptions, evidence, constraints, hypotheses. It updates ELCS files as the conversation evolves.

# elcs/state/current.json
{
  "assumptions": [...],
  "hypotheses": [...],
  "constraints": [...]
}

üí° Tip: After long sessions, it's good to remind the agent: "Update the ELCS state files with what we've learned so far." This keeps everything fresh and recoverable.

Step 3 ‚Äî Consult the Lenses

The 7 lenses evaluate your project from different perspectives ‚Äî philosophy, data science, safety, topology, math, systems engineering, and product/UX.

"Apply the 7 lenses to our current
state and identify gaps."

Lenses aren't just for the beginning. Use them anytime ‚Äî when you're stuck, need fresh ideas, troubleshooting a problem, or want to pressure-test a decision:

"Run the safety and product lenses
on this architecture decision."

"I'm stuck on the data model ‚Äî
what do the lenses suggest?"

Step 4 ‚Äî Earn Goals & Build

Goals pass through 6 quality gates and become WorkTokens. The agent executes work in loops, checkpoints regularly, and you review at natural breakpoints.

"Check elcs/tokens/open/ and
recommend the highest priority task."

Your plans keep evolving as you learn. New evidence updates the state, which reshapes priorities. "Done" is relative to your needs ‚Äî ELCS adapts with you, not against you.

A project might start as a quick prototype and grow into a full product. ELCS scales the process to match ‚Äî always tracking where you are and what matters next.

The one rule: Write to files, not just chat. If your session ends, ELCS artifacts survive. That's the entire philosophy in one sentence.
