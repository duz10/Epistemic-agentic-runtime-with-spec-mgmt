## How to Contribute

### Reporting Issues

1. Check [existing issues](https://github.com/duz10/Epistemic-agentic-runtime-with-spec-mgmt/issues) first
2. If not found, [open a new issue](https://github.com/duz10/Epistemic-agentic-runtime-with-spec-mgmt/issues/new)
3. Include:
   - Clear description of the problem or suggestion
   - Steps to reproduce (for bugs)
   - Your environment (OS, AI tool, etc.)

### Submitting Changes

We welcome pull requests! Here's how:

#### 1. Fork the Repository

Click the **Fork** button on GitHub to create your own copy.

#### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/Epistemic-agentic-runtime-with-spec-mgmt.git
cd Epistemic-agentic-runtime-with-spec-mgmt
```

#### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

Use descriptive branch names:
- `feature/add-new-lens` 
- `fix/install-script-windows`
- `docs/improve-quickstart`

#### 4. Make Your Changes

- Follow existing code style and conventions
- Update documentation if needed
- Test your changes

#### 5. Commit Your Changes

```bash
git add .
git commit -m "Brief description of changes"
```

Write clear commit messages:
- ‚úÖ `Add support for Zed editor bootstrap`
- ‚úÖ `Fix Windows path handling in install.ps1`
- ‚ùå `Fixed stuff`
- ‚ùå `WIP`

#### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

#### 7. Open a Pull Request

1. Go to your fork on GitHub
2. Click **Compare & pull request**
3. Fill in the PR template:
   - What does this PR do?
   - Why is it needed?
   - How was it tested?
4. Submit!

### What Happens Next

1. A maintainer will review your PR
2. They may request changes or ask questions
3. Once approved, your PR will be merged
4. You'll be credited in the project! üéâ
